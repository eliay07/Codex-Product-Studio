# 闭环评审与 QA 清单

## 产品闭环

- [ ] 目标用户具体。
- [ ] 产品定位、最高频场景和用户成功瞬间已明确。
- [ ] 核心任务用动词描述。
- [ ] 已选择通用产品表面，例如 entry、workflow、management、editor、reader、asset、monitoring 或 settings。
- [ ] 页面结构匹配所选产品表面，没有把所有场景都套成同一种 dashboard、landing page 或卡片流。
- [ ] 功能已按使用频率、用户价值、业务价值和风险排序。
- [ ] 第一、第二、第三步清晰。
- [ ] 已选择合理动线模型，例如线性向导、hub and spoke、列表-详情、画布-工具栏、搜索-结果或仪表盘-下钻。
- [ ] 页面入口清晰。
- [ ] 成功后有明确去向。
- [ ] 失败后有恢复路径。
- [ ] 本版本不做什么已写清楚。

## 信息架构

- [ ] 顶层导航稳定且不过多。
- [ ] 导航排序符合高频优先、高价值优先、低频折叠的逻辑。
- [ ] 页面标题说明当前目的。
- [ ] 页面标题短而像产品名/任务名，不像需求说明句。
- [ ] 用户知道自己在哪里。
- [ ] 页面主区域只承载一个主目标。
- [ ] 页面区块排序符合高频任务、当前状态、关键输入/输出、辅助工具、低频设置的优先级。
- [ ] 复杂流程拆成步骤或渐进披露。
- [ ] 标签使用用户语言，不用内部黑话。
- [ ] 主要 tab / step 有图标、状态或视觉指引，不只靠文字。

## 2C 首屏和转化

- [ ] 如果是 2C 产品，首屏 3 秒内能看懂产品用途。
- [ ] 首屏展示最高频、最吸引人的动作或结果预期。
- [ ] 首屏有明确开始路径，例如输入、上传、试用、生成、搜索或购买。
- [ ] 首屏没有堆功能大全、内部策略、大段理念或低频设置。
- [ ] 首屏视觉能展示结果或价值，不只靠文字说明。

## 按钮和交互

- [ ] 每个视图最多一个 primary action。
- [ ] 次要动作没有抢主按钮。
- [ ] 危险动作有确认、撤销或后果说明。
- [ ] 页面级、对象级、批量、筛选、危险动作的位置各有依据。
- [ ] 搜索、筛选、排序靠近它们影响的内容。
- [ ] 批量操作只在选择对象后出现。
- [ ] 按钮文案使用动词。
- [ ] 同一意图没有多个文案。
- [ ] 每个动作都有 loading/success/error。
- [ ] disabled 状态解释原因或有明显上下文。

## 前端质量

- [ ] 复用现有组件和 tokens。
- [ ] 已建立色彩角色表，包含 primary、secondary、accent、中性色、语义色和状态色。
- [ ] 页面代码没有散落裸 hex；裸色值集中在 token/theme 文件中。
- [ ] 色彩比例接近 60% 中性底、30% 层级分区、10% 品牌强调。
- [ ] 页面没有被单一色相统治，也没有随机渐变、发光和高饱和色堆叠。
- [ ] 字体层级、行高、字号和字重有统一规则。
- [ ] 布局使用稳定网格、对齐、gap、max-width 和响应式断点。
- [ ] 布局在移动端和桌面都可用。
- [ ] 文本不溢出、不重叠。
- [ ] focus ring 可见。
- [ ] 表单有 label 和错误提示。
- [ ] 对比度满足 WCAG AA。
- [ ] 空状态说明下一步。
- [ ] 图标按钮有可访问名称或 tooltip。
- [ ] 如果是官网、2C 页面或品牌化产品，已规划 logo、背景、图标、空状态、社交图和产品截图等统一素材库。
- [ ] 素材风格、色卡、线条、圆角、光影和构图一致。
- [ ] 页面文案没有泄露 PRD、实现策略、设计原则或“我们要/不要怎么做”。
- [ ] 已列出所有可见文案，并把内部说明移到文档、注释或交付说明。
- [ ] 已检查哪些长句可以改成图标、badge、进度、checklist、tooltip 或帮助抽屉。
- [ ] 示例、模板和规则没有使用其他项目的真实产品名、业务对象、导航标签、页面文案或品牌风格。

## 后端契约

- [ ] 每个可见动作对应接口或本地状态。
- [ ] 请求、响应、错误码明确。
- [ ] 权限规则明确。
- [ ] 校验规则前后端一致。
- [ ] 长任务有 job 状态。
- [ ] 删除/发布/扣费/覆盖类操作防重复提交。
- [ ] 重要操作有审计或日志。

## AI 功能

- [ ] 输入上下文明确。
- [ ] 生成中可见且可取消或等待。
- [ ] 结果进入预览或待确认，不直接覆盖。
- [ ] 支持编辑、重试、接受、拒绝。
- [ ] 有失败、限流、超时处理。
- [ ] 有成本、隐私或版权风险提示策略。

## 验证

- [ ] 跑过项目 lint/test/build 或说明无法运行。
- [ ] 手动走通 happy path。
- [ ] 手动走通过空、错、无权限、加载中。
- [ ] 检查移动端核心任务。
- [ ] 检查键盘操作。
- [ ] 截图或本地浏览器验证过关键页面。
- [ ] 对关键页面做过视觉截图检查，确认色彩舒适、排版规整、控件状态可区分。
- [ ] 使用 `check-product-loop.sh` 检查过产品闭环和可见文案风险。
