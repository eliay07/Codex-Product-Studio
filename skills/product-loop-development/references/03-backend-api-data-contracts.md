# 后端 API 与数据契约

## 一句话原则

前端每个可见动作都要对应明确的数据来源、状态变化、错误处理和权限规则。

## 实体定义模板

```text
实体:
用途:
主键:
字段:
枚举:
关系:
创建者:
可见范围:
可编辑范围:
生命周期:
软删除/硬删除:
审计日志:
```

```text

审计事件:

```

## 状态映射

后端错误不能直接裸露给用户。需要映射成产品语义：

| 后端情况 | 前端反馈 | 用户下一步 |
|---|---|---|
| 401 未登录 | 要求登录 | 去登录 |
| 403 无权限 | 说明无权限 | 联系管理员或切换空间 |
| 404 不存在 | 对象不存在或已删除 | 返回列表 |
| 409 冲突 | 数据已被更新 | 刷新、合并或重试 |
| 422 校验失败 | 定位字段错误 | 修改输入 |
| 429 限流 | 告知等待时间 | 稍后重试 |
| 5xx 服务错误 | 说明系统失败 | 重试或保存草稿 |

## 表单约束

每个表单必须有：

- 默认值来源。
- 字段级校验。
- 表单级校验。
- 提交中状态。
- 提交失败恢复。
- 离开页面时的未保存提醒。
- 成功后的去向。

## 列表和详情

列表必须定义：

- 搜索字段。
- 筛选项。
- 排序。
- 分页或无限滚动。
- 空列表。
- 空搜索结果。
- 批量操作。
- 权限差异。

详情必须定义：

- 对象标题。
- 关键元信息。
- 主动作。
- 状态变化历史。
- 关联对象。
- 删除或归档策略。

## AI 功能契约

AI 功能不能只写一个“生成”按钮。必须定义：

- 输入：用户提供什么、系统自动带什么上下文。
- 约束：长度、格式、禁用内容、价格或次数限制。
- 生成中：进度、取消、超时。
- 结果：草稿、预览、引用来源、置信度或待确认点。
- 人工控制：接受、编辑、重试、撤销。
- 持久化：何时保存，保存原始输入还是结果。
- 风险：幻觉、版权、隐私、越权、成本暴涨。

## 工程实现约束

- 不在前端硬编码业务权限。
- 不让同一业务状态在前后端使用不同枚举名。
- 写接口前先定错误码和前端展示。
- 删除、发布、扣费、覆盖类接口需要幂等或防重复提交。
- 长任务需要 job id、轮询或事件流、取消和失败重试。
- 重要操作写审计事件。
- 涉及用户内容和 AI 输入输出时，明确数据留存策略。
