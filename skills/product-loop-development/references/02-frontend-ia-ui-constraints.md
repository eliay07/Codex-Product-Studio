# 前端信息架构与 UI 约束

## 信息架构

### 导航

- 导航排序必须先看用户任务频率和优先级；最高频、最高价值的入口靠前，低频设置和管理类入口靠后或折叠。
- 顶层导航保持稳定，默认 5 到 7 个以内。
- 同一层级不要混合不同组织原则，例如“项目、设置、按团队、报表”硬塞在一起。
- 工作型产品优先让用户知道位置：active nav、页面标题、面包屑或步骤条至少有一个。
- 不要把核心功能藏在卡片角落的弱按钮里。

### 页面结构

推荐布局：

```text
页面标题区: 位置 + 页面目标 + 主动作
状态摘要区: 当前对象/进度/风险
主任务区: 用户完成核心任务
辅助区: 筛选、历史、说明、设置
反馈区: toast、inline error、empty state
```

页面区块排序必须遵循：

```text
高频任务 > 当前状态 > 关键输入/输出 > 辅助工具 > 低频设置 > 帮助说明
```

复杂任务使用步骤：

```text
Step 1 输入/选择
Step 2 预览/校验
Step 3 确认/提交
Step 4 结果/下一步
```

## 按钮和动作

### 动作层级

| 层级 | 用途 | 约束 |
|---|---|---|
| Primary | 推进主流程 | 每个视图默认只放一个 |
| Secondary | 辅助但重要 | 不抢主按钮视觉 |
| Tertiary | 低频操作 | 用文字按钮、图标按钮或菜单 |
| Destructive | 删除、覆盖、清空 | 需要确认、撤销或明确后果 |

### 按钮文案

- 用动词开头：`生成分析`、`保存草稿`、`发布内容`。
- 不用模糊词：`确定`、`提交`、`下一步`，除非上下文足够清晰。
- 同一个意图全项目使用同一个文案。
- 不要让两个按钮表达同一件事，例如 `开始` 和 `立即生成` 同时出现。

### 按钮位置

- 页面级主动作放在标题区右侧、主任务区末尾或移动端底部固定动作区，不能随机出现在卡片角落。
- 对象级动作靠近对象，例如列表行操作、卡片菜单、详情页操作区。
- 搜索、筛选、排序靠近被影响的列表或结果区。
- 批量操作只在用户选择对象后出现。
- 危险动作和主动作必须视觉隔离，不能相邻摆放成同等权重。
- 同一层级按钮顺序要稳定，不能不同页面一会儿左主右次、一会儿右主左次。

### 图标按钮

- 图标按钮必须有可访问名称或 tooltip。
- 常见工具动作优先用熟悉图标：保存、下载、删除、搜索、过滤、更多。
- 不手绘临时 SVG 图标，优先用项目已有 icon 库。

## 状态矩阵

每个交互组件至少考虑：

```text
default
hover
focus
active
disabled
loading
success
error
empty
permission-denied
```

表单还要考虑：

```text
pristine
dirty
validating
invalid
submitting
submit-failed
submitted
```

AI 功能还要考虑：

```text
waiting-for-input
generating
partial-result
needs-review
accepted
rejected
retrying
rate-limited
```

## 视觉系统

- 先复用项目已有组件和 tokens。
- 没有设计系统时，建立最小 tokens：颜色、字号、间距、圆角、边框、阴影、动效。
- 工作型 SaaS、CRM、后台、运营工具应安静、密集、可扫描，不做营销式大 hero。
- 卡片只用于重复项目、modal 或明确需要框住的工具；不要卡片套卡片。
- 色彩只服务状态、层级和品牌，不用装饰性渐变堆气氛。
- 一个页面只保留一个主强调色。
- 小控件和按钮文字必须稳定，不因 hover/loading 改变尺寸。

## 可访问性和响应式

- 只有一个 `h1`，标题层级顺序清晰。
- 表单控件必须有 label。
- 键盘 tab 顺序符合视觉顺序。
- focus ring 可见。
- 对比度满足 WCAG AA。
- 移动端主动作不被挤到屏幕外。
- 文本不与按钮、卡片、表格列重叠。

## 前端交付前必须问

- 用户一眼知道第一步在哪里吗？
- 页面上哪个按钮最重要是显而易见的吗？
- 次要按钮有没有抢主按钮？
- 点击每个按钮之后有没有反馈？
- 空状态有没有告诉用户下一步？
- 错误状态有没有给出可执行恢复路径？
- 移动端是否还能完成完整任务？
- 页面上的可见文案是否都是用户完成任务所需，而不是 PRD、设计原则、实现策略或内部说明？
- 有没有出现“我们要/不要怎么做”“本版本不做”“产品闭环”“验收标准”等本应写进文档的文字？
