# 交互深浅层级与渐进披露规范

## 一句话原则

成熟产品不是把所有功能铺在一个页面，而是先建立交互层级：高频主任务默认可见，辅助信息可收起，低频能力进菜单，高复杂度内容进抽屉/详情/浮层，跨页面动作进命令入口。

## 为什么必须先做交互层级

Codex 生成前端时常见问题：

- 把所有功能、数据、设置、预览和解释同屏铺开。
- 所有模块同等大小、同等颜色、同等按钮权重。
- 没有进入深浅，用户不知道先看哪里、先点哪里。
- 预览、历史、来源、风险、参数、日志和帮助抢占主任务空间。
- 低频操作和高频操作在同一层，导致页面像功能清单而不是产品。
- 需要沉浸编辑的产品仍停留在 dashboard 式平铺页面。

解决方式不是“少放功能”，而是给功能分层、收纳和按需展开。

## 调研来源

这些高质量开源项目提供了可复用的交互方向：

| 项目 | 适合借鉴的交互问题 | 使用方式 |
|---|---|---|
| [Radix UI Primitives](https://github.com/radix-ui/primitives) | Dialog、Popover、Dropdown、Tabs、Accordion 等可访问基础交互 | 低层组件和可访问状态 |
| [Floating UI](https://github.com/floating-ui/floating-ui) | tooltip、popover、floating panel 的定位和碰撞处理 | 悬浮层、上下文菜单、气泡 |
| [shadcn/ui](https://github.com/shadcn-ui/ui) | Sheet、Dialog、Command、Tabs、Drawer、Resizable 等组合式 UI | 快速组合成熟应用壳 |
| [cmdk](https://github.com/dip/cmdk) | 命令菜单和搜索式动作入口 | 跨页面低频动作、快速跳转、操作搜索 |
| [react-resizable-panels](https://github.com/bvaughn/react-resizable-panels) | 可调整工作区面板 | IDE / Studio / 编辑器布局 |
| [Vaul](https://github.com/emilkowalski/vaul) | 移动端和桌面抽屉体验 | 详情、确认、辅助面板 |
| [React Spectrum](https://github.com/adobe/react-spectrum) | 自适应、可访问、稳健体验 | 键盘、焦点、状态、国际化 |
| [xyflow / React Flow](https://github.com/xyflow/xyflow) | 节点式流程和工作流编辑 | 动线、自动化、任务依赖图 |
| [tldraw](https://github.com/tldraw/tldraw) | infinite canvas 应用 | 画布型创作、空间化组织复杂对象 |
| [TanStack Router](https://github.com/TanStack/router) | 类型安全路由和嵌套路由 | 深层详情页、保留上下文的路由层级 |

不要照搬这些项目的视觉风格或业务结构。只借鉴交互模式、组件职责和状态处理。

## 交互层级模型

每个页面先分 5 层，再写 UI：

```text
L0 固定壳层: 主导航、当前对象、阶段进度、全局状态。
L1 默认任务层: 当前页面唯一主任务和一个 primary action。
L2 上下文层: 预览、来源、历史、风险、建议、详情，可收起或可调整宽度。
L3 临时操作层: 筛选、更多、快捷动作、tooltip、popover、dropdown、command menu。
L4 深层详情层: 完整详情、日志、版本、参数、设置、危险操作，进入抽屉、详情页或 modal。
```

页面默认只应暴露 L0 + L1 + 必要的 L2。L3/L4 必须按需出现。

## Entry 到确认的后置问卷模式

AI 产品、生成类工具、调研工具和产品启动工具常见错误，是把问卷、参数、分析维度和流程说明直接铺在首屏。成熟入口应该先让用户完成一个低成本表达，再在用户明确发送、开始或继续后，展示必要确认。

默认顺序：

```text
1. Entry: 一句短价值表达 + 主输入/主动作 + 少量示例或结果预期。
2. Submit: 用户点击发送、开始、生成、继续或选择示例。
3. Clarify: 系统基于输入预填轻量问卷、关键确认或候选项。
4. Confirm: 用户修改、补充、确认或跳过非必要项。
5. Workspace: 进入动线、分层、编辑、预览、输出或执行工作台。
```

首屏可以出现模式切换、示例入口、附件、最近记录和结果预期，但不默认展示完整问卷、全部参数、方法论解释、检查清单或 PRD 式流程说明。问卷如果必要，应作为 L4 短流程 modal、drawer、step 或独立确认页出现，并保留“确认继续”和“稍后补充/跳过”路径。

后置问卷适用：

```text
从一句灵感生成产品
AI 调研 / AI 分析
AI 内容创作
需求澄清
Agent Brief 生成
复杂任务启动
```

检查句：

```text
用户还没发送前，页面是否已经要求他填大量配置？
问卷是基于用户输入预填的吗？
用户是否能先表达需求，再确认系统理解？
确认完成后是否进入工作台，而不是继续停留在表单页？
```

## 默认层应该放什么

默认层只放：

```text
当前位置
当前对象
当前任务
当前状态
最高频动作
必要输入
必要预览
下一步
```

默认层不放：

```text
完整来源
全部历史
高级参数
低频设置
调试日志
所有候选
所有解释
所有二级动作
```

## 收起和浮层策略

### 侧栏

适合：

```text
主导航
对象列表
文件树
草稿列表
阶段目录
```

要求：

```text
桌面可收起
收起后保留图标或状态点
移动端可转为抽屉或顶部入口
收起后必须保留 40-64px rail、handle、tab、floating trigger 或命令入口
展开时优先恢复上一次宽度和状态，不要只恢复到最小值
恢复入口必须可键盘聚焦，并提供 aria-label 或 tooltip
```

禁止：

```text
把面板宽度收为 0，同时把恢复按钮也隐藏。
用遮罩或浮层挡住全部导航，却不给明确关闭和返回路径。
只改 CSS 宽度，不处理 focus、aria、hover、键盘和状态记忆。
```

### 右侧面板

适合：

```text
预览
属性
风险
来源
方法细节
版本
检查项
```

要求：

```text
可关闭或可调整宽度
不抢主编辑区
打开后保持当前对象上下文
```

### Sheet / Drawer

适合：

```text
详情
确认
辅助配置
移动端二级内容
不需要离开当前页的深层信息
```

要求：

```text
有明确标题
有返回/关闭
关闭后不丢失主任务状态
```

### Popover / Dropdown / Tooltip

适合：

```text
短说明
快捷选择
更多动作
局部筛选
对象级菜单
```

要求：

```text
内容短
靠近触发对象
不能承载长流程
键盘可关闭
```

### Command Menu

适合：

```text
跨页面跳转
低频动作
搜索功能
快速执行
高级用户快捷入口
```

要求：

```text
不是主路径替代品
动作命名清楚
支持搜索和分组
危险动作不能直接执行
```

### Modal

适合：

```text
必须打断的确认
高风险动作
短流程聚焦输入
```

不适合：

```text
长表单
复杂编辑
频繁查看详情
普通预览
```

## 产品表面对应的交互结构

| 产品表面 | 默认结构 | 深层交互 |
|---|---|---|
| Entry surface | 主输入 / 主动作 / 少量示例；需要问卷时发送后再出现 | 示例详情用 drawer，低频设置用 popover，确认问卷用 modal / drawer / step |
| Workflow surface | 步骤轨道 / 当前步骤 / 下一步 | 来源、参数、日志放右侧面板或 drawer |
| Management surface | 列表 / 搜索 / 筛选 / 对象状态 | 批量操作选中后出现，详情进 drawer 或详情页 |
| Editor surface | 中央编辑区 / 工具栏 / 保存状态 | 属性、历史、预览、素材放可收起面板 |
| Reader surface | 内容主体 / 目录 / 核心结论 | 引用、批注、版本、导出放侧栏或 floating action |
| Asset surface | 分类 / 网格 / 预览 | 标签、来源、导入详情进 drawer |
| Monitoring surface | 状态摘要 / 异常列表 | 下钻详情、日志、处理动作进详情页或 drawer |
| Settings surface | 分组设置 / 保存取消 | 高风险动作进危险区和确认 modal |

## 工作台 / 编辑器状态模型

工作台、编辑器、创作者工具和 AI 生成工具必须明确这些状态，不要默认全部展开：

```text
Default / hub: 未选择对象。展示最近记录、主入口、空状态下一步；不自动打开深层编辑器。
Object selected: 用户选中一条记录。展示摘要、预览、继续按钮或右侧 inspector。
Editing active: 用户确认打开或继续。主编辑区占最大面积，相关 AI 工具靠近对象。
Focus mode: 用户想专注。侧栏、列表、辅助区可折叠，但恢复入口持续可见。
Utility open: 用户打开日志、素材、参数、版本、来源或帮助。它们进入 drawer / bottom panel / floating popover。
Output / review: 用户生成、保存、导出或发布前，展示预览、状态和确认。
Error / recovery: 失败时保留上下文，给重试、撤销、返回或查看日志入口。
```

默认态规则：

- 工作台默认不要自动选中最近对象并直接进入编辑，除非用户明确要求“打开上次工作”。
- 默认态必须有一个清楚下一步：新建、打开最近、从库中选择、导入或继续。
- 最近对象可以展示，但按钮文案必须是“打开/继续”，不能自动替用户进入。
- 没有对象时，主编辑区显示空状态和主入口，不要显示 disabled 的巨大表单假装可编辑。

进入深层规则：

```text
点击列表项 / 打开草稿 / 继续编辑 / 创建后打开 / 从预览确认 / 快捷搜索选中
```

创建、生成或导入成功后，不一定立即跳入编辑器：

- 轻量任务：成功后给“打开结果”按钮。
- 强连续任务：成功后进入下一步，但提供返回和撤销。
- 高风险任务：先预览，再确认进入或覆盖。

工作台面板分工：

| 层 | 放什么 | 不放什么 |
|---|---|---|
| Activity rail | 大模块、恢复入口、全局状态 badge | 长文本、复杂设置 |
| Sidebar / list | 对象集合、搜索、筛选、最近记录 | 当前对象的全部详情 |
| Main editor | 正在编辑的正文、画布、脚本、预览 | 日志、来源长列表、全局设置 |
| Inspector / drawer | 当前对象详情、参数、来源、版本、风险 | 多个对象的管理列表 |
| Bottom panel | 日志、生成过程、错误、上传队列 | 长期导航和主内容 |
| Floating action | AI 助手、快捷生成、评论、帮助 | 高频主流程和危险动作 |
| Command palette | 跳转、搜索、低频命令、设置入口 | 必须显眼的主动作 |

## 实现前必须产出交互层级图

每次从零做前端、重构页面或用户反馈“功能平铺、交互不舒服、没有层次”时，先输出：

```text
页面:
主任务:
L0 固定壳层:
L1 默认任务层:
L2 可收起上下文:
L3 临时浮层:
L4 深层详情:
默认打开:
默认收起:
移动端替代:
键盘入口:
危险动作位置:
可折叠区域:
恢复入口:
默认是否自动打开对象:
进入编辑的明确动作:
```

## 组件选择建议

React / Next.js 项目优先：

```text
shadcn/ui: Sheet、Dialog、Command、Tabs、Resizable、Popover。
Radix UI: 可访问基础组件，适合自定义设计系统。
Floating UI: 复杂浮层定位、tooltip、上下文菜单。
cmdk: 命令菜单。
react-resizable-panels: IDE / Studio 型工作区。
Vaul: 抽屉，尤其适合移动端。
React Flow / xyflow: 节点式动线、流程和依赖关系。
Tldraw: 画布型创作和空间化组织。
React Spectrum: 对可访问和自适应要求高的产品。
TanStack Router: 深层路由和上下文保留。
```

组件不是越多越好。只有当一个信息或动作确实需要降级、悬浮、收起、搜索或空间化时，才引入对应组件。

## 检查清单

交付前必须检查：

```text
是否有交互层级图。
默认层是否只突出一个主任务。
问卷、参数和确认项是否在用户发送后才出现，而不是默认铺在首屏。
低频动作是否进入菜单、命令入口、抽屉或详情页。
辅助信息是否可收起、可关闭或可调整宽度。
预览是否贴近当前对象，而不是远离主任务。
复杂编辑是否给主编辑区最大空间。
移动端是否把侧栏转成抽屉或顶部入口。
浮层是否有关闭、焦点和键盘路径。
危险动作是否被隔离到深层、确认或危险区。
用户不读大段文字是否仍知道下一步。
默认态是否避免自动进入深层编辑。
每个折叠面板收起后是否仍能一键恢复。
展开时是否恢复上一次宽度、高度或状态。
抽屉、浮层和遮罩是否不会阻断导航、关闭或恢复路径。
主编辑区是否始终是编辑态最大面积。
```

## 给 Codex 的硬约束句

```text
在写前端前，先产出交互层级图。不要把所有功能平铺在一个页面。默认层只放当前主任务和一个主动作；AI 产品、生成类工具、调研工具和产品启动工具的问卷、参数、确认项必须在用户发送/开始后再出现，不要默认铺在首屏；来源、历史、参数、预览、风险、设置和低频动作必须按重要性进入可收起侧栏、右侧面板、Sheet/Drawer、Popover/Dropdown、Command Menu、详情页或 Modal。移动端必须说明侧栏和浮层如何转换。
```
