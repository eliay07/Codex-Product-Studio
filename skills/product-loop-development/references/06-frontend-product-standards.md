# 前端成熟产品规范

## 一句话原则

前端不是把需求说明展示出来，而是把用户任务、业务状态和产品判断转译成可操作界面。

## 成熟产品感

- 页面先服务任务，不先展示观点。
- 页面功能排序必须体现产品判断：高频、高价值、低学习成本的入口优先出现，低频和高风险功能降级。
- 首屏必须让用户知道：我在哪里、能做什么、下一步是什么。
- 2C 产品首屏必须让用户快速理解用途、看到最高频动作或结果预期，并能立即开始。
- 用户不应依赖阅读长段说明理解流程；优先用图标、tab、步骤、状态和视觉层级提示。
- 工作型产品优先清晰、安静、密集、可扫描；不要做营销式大 hero、装饰性卡片堆叠和过度渐变。
- 每个页面只保留一个主目标；其他内容降级为辅助区、折叠区、菜单或详情页。
- 每个页面必须有交互深浅层级；默认层服务主任务，辅助层可收起，临时操作进浮层，深层设置和风险动作进抽屉、详情页或确认弹窗。
- 组件尺寸稳定，hover、loading、错误文案不得造成布局跳动。
- 文案、按钮、图标、表格列、表单字段在移动端和桌面端都不能重叠或溢出。

## 信息架构

页面结构优先顺序：

```text
位置感: 标题、面包屑、active nav、步骤条
最高频入口: 输入、上传、创建、搜索、生成、试用
任务入口: 主动作、输入区、导入区或选择区
任务主体: 表单、列表、编辑器、结果区、预览区
状态反馈: loading、empty、error、success、permission
辅助信息: 历史、说明、设置、筛选、帮助
```

不要把“为什么这样设计”“我们不做什么”“实现方式”放在任务主体。它们属于 PRD、设计说明或交付说明。

## 交互层级

成熟产品要先判断功能的进入深浅，而不是把所有东西摊开：

```text
L0 固定壳层: 导航、当前位置、当前对象、全局状态。
L1 默认任务层: 用户当前要完成的唯一主任务和主动作。
L2 可收起上下文: 预览、来源、历史、风险、属性、版本。
L3 临时浮层: 筛选、更多动作、tooltip、popover、dropdown、command menu。
L4 深层详情: 高级参数、日志、完整详情、设置、危险动作。
```

默认页面只展示 L0 + L1 + 必要 L2。不要让来源、候选、参数、预览、历史、风险、设置和所有动作同屏同权重出现。

交互结构建议：

```text
Entry / guided flow: 默认主输入或当前步骤；示例、参数、帮助进抽屉或 popover。
Editor / studio: 中央编辑区最大；对象列表、属性、历史、预览、日志可收起。
Management: 默认列表和搜索；批量操作只在选中后出现；详情进 drawer 或详情页。
Reader / report: 默认核心内容；引用、方法、风险、版本用侧栏或展开层。
Monitoring: 默认异常和下一步；日志和明细下钻。
Settings: 默认常用设置；危险动作放危险区和确认 modal。
```

## 通用产品表面

成熟前端先匹配任务类型，再选择结构。常见表面包括：

```text
Entry surface: 开始最高频任务，核心是主输入/主动作和结果预期。
Workflow surface: 连续步骤或长任务，核心是进度、状态流和失败恢复。
Management surface: 管理对象集合，核心是搜索、筛选、列表、批量和详情入口。
Editor surface: 编辑单个对象，核心是工具栏、画布/表单、预览、保存、撤销。
Reader surface: 阅读或审阅结果，核心是目录、进度、引用、批注、导出。
Asset surface: 浏览复用资产，核心是分类、预览、标签、导入和收藏。
Monitoring surface: 监控状态和异常，核心是摘要、趋势、异常列表和下钻。
Settings surface: 配置低频选项，核心是分组、权限、保存和危险区域。
```

不要把某个已有产品的页面、文案、导航或业务对象作为固定范例写进新项目。只能复用抽象结构和质量检查项。

## 设计系统

视觉风格必须先由产品定位和目标用户推导：

```text
目标用户: 新手 / 专业人士 / 创作者 / 消费者 / 企业客户
使用场景: 高频工作 / 轻娱乐 / 决策购买 / 创作编辑 / 数据监控
情绪目标: 安全 / 高效 / 灵感 / 信任 / 兴奋 / 克制
视觉性格: 专业 / 温暖 / 游戏化 / 科技 / 高级 / 活泼
信息密度: 高 / 中 / 低
```

没有现成设计系统时，先建立最小 tokens：

```text
颜色: primary、secondary、accent、background、surface、text、muted、border、success、warning、danger、info
字号: page-title、section-title、body、caption、button
间距: 4、8、12、16、24、32
圆角: 4 或 8，除非项目已有规则
阴影: 少用，只表达层级
动效: 只用于反馈、过渡和状态变化
```

色彩不能只写一组“看起来不错”的 hex。必须定义角色、用途和状态：

```text
色阶: 50 / 100 / 200 / 300 / 400 / 500 / 600 / 700 / 800 / 900
语义: success / warning / danger / info
状态: hover / active / focus / selected / disabled / loading
模式: light / dark，或明确本版本只做 light
比例: 60% 中性底 + 30% 层级和分区 + 10% 品牌强调
```

涉及色彩、版式、视觉舒适度时，先按 `12-color-typography-layout-systems.md` 输出色彩角色表、字体层级表、布局网格表和状态映射表。

禁忌：

- 不用大面积紫蓝渐变、玻璃拟态、发光球、bokeh、装饰 orb 作为默认风格。
- 不让页面被单一色相统治；避免全紫、全蓝、全棕、全米色这类一色到底的界面。
- 不在组件里散落裸 hex；裸色值只允许集中出现在 token 定义文件。
- 不用通用 AI dashboard 套壳：一堆彩色卡片、空洞指标、泛泛 hero、随机 icon。
- 不做卡片套卡片。
- 不把说明文字堆成“教程页”，除非用户正在 onboarding。

## 控件选择

- 工具动作优先用图标按钮加 tooltip：保存、下载、搜索、过滤、更多、撤销、重做。
- 模式选择用 segmented control 或 tabs。
- 主要流程 tab 必须有图标或视觉标识；仅文字 tab 容易显得像文档目录。
- 布尔设置用 switch 或 checkbox。
- 数字范围用 slider、stepper 或 number input。
- 多选/筛选用 checkbox group、combobox 或 menu。
- 危险操作用明确的 destructive 样式，并提供确认、撤销或后果说明。

## 素材库

官网、2C 产品和品牌化工具必须先规划素材库：

```text
logo / wordmark
app icon / favicon
hero background
feature icons
empty state illustration
onboarding illustration
social preview image
loading / success / error visuals
product screenshots or mockups
```

素材必须共享色卡、线条、圆角、光影和构图语言。不要让背景图、logo、图标和空状态像来自不同产品。

## 状态覆盖

每个页面至少覆盖：

```text
首次进入
已有数据
空数据
加载中
提交中
成功
失败
无权限
移动端
键盘焦点
```

AI 功能额外覆盖：

```text
等待输入
生成中
部分结果
需要人工确认
已接受
已拒绝
重试中
限流或失败
```

## 交付前视觉检查

- 页面上所有可见文案都像真实产品，不像 PRD 摘录。
- 页面有统一色彩模式和 token，主色、辅助色、状态色职责清楚。
- 版式有稳定网格、对齐和留白，不像随手拼贴。
- 功能排序、按钮位置和首屏内容能说明产品判断，而不是随机摆放。
- 官网或 2C 页面有统一素材库策略，不是临时拼图。
- 主按钮唯一且显眼。
- 次要动作没有抢主按钮。
- 空状态给出下一步。
- 错误状态给出恢复路径。
- 页面没有无意义的装饰文字、解释型标语或内部策略。
- 浏览器截图中没有明显重叠、截断、错位、空白大洞。
- 页面结构与产品表面匹配，没有因为套用通用 AI 模板而出现无关指标、无关卡片或无关 hero。
